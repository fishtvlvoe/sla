# SLA Release — 推送 PR + CI/CD 觸發

代碼 Review 通過後，一鍵推送、開 PR、觸發 CI。

## 使用方式

```
/sla:release           # 推送當前分支並開 PR
/sla:release --draft   # 開草稿 PR（尚未 Review 完）
```

## 執行流程

```
確認 Code Review 通過（.planning/REVIEW.md 狀態為 passed）
    ↓
確認所有測試通過
    ↓
git push origin [current-branch]
    ↓
自動生成 PR 描述（雙語）
    ↓
gh pr create --title "..." --body "..."
    ↓
GitHub Actions 自動觸發
    ↓
通知用戶 PR URL
    ↓
自動進入 /sla:status 監控 CI 狀態
```

## PR 描述模板（雙語）

```markdown
## Summary / 摘要

<!-- English -->
- [Feature/Fix description in one line]
- Key changes made
- Breaking changes (if any)

<!-- 中文 -->
- [功能/修復的一句話描述]
- 主要變動說明
- 破壞性變更（如有）

## Test Plan / 測試計畫

- [ ] Unit tests passing
- [ ] E2E tests passing (Playwright)
- [ ] Manual verification on local environment

## Related Issues / 相關議題

Closes #[issue number]

---
🤖 Generated by [Smart LLM Agents (SLA)](https://github.com/fishtvlvoe/sla)
```

## 安全檢查（推送前）

| 檢查項目 | 說明 |
|---------|------|
| 無 .env 或敏感檔案在暫存區 | `git diff --cached` 確認 |
| 測試全部通過 | 最後一次測試結果為綠燈 |
| Review 狀態為 passed | `.planning/REVIEW.md` 確認 |
| 非 main/master 分支 | 防止直接推到主分支 |

**若安全檢查失敗 → 停止推送，回報原因**

## GitHub Actions 觸發

推送後 GHA 自動執行（需用戶已在專案中設置 `.github/workflows/sla-ci.yml`）：

```
PR opened
    ↓
GHA: sla-ci.yml 執行
  ├── 安裝依賴
  ├── 跑單元測試
  ├── 跑 Playwright E2E
  └── 回報結果
    ↓
/sla:status 自動監控結果
```

## GSD 連動

- 執行前確認 GSD 狀態為 review-passed
- 推送後更新 GSD 狀態為 released
- 寫入 `.planning/RELEASE.md`（PR URL + 時間戳）

## 注意事項

- **禁止** `git push --force`（除非用戶明確要求並確認）
- **禁止** 直接推到 main/master
- PR 標題 < 70 字元
- 必須附 PR 描述，不能空白
